К сожалению, мне не удалось запустить тестирование из за ошибки при запуске симуляции. Независимо от симулятора 
(я пробовал ModelSim, Vivado 2025.2 и Verilator) - симуляция падает из за типа type_scr1_ialu_cmd_sel_e. 
Ни один из симуляторов не может объединить шину этого типа моего нтерфейса с тестируемым модулем. Пробовал разные варианты решения проблемы,
но ничего не помогло. Несмотря на это, предоставляю решение в том виде, до которого удалось его довести без реальной симуляции.

Кодовое покрытие: оценку выполнить не удалось ввиду описанной выше проблемы. Тесты проверяют каждую из поддерживаемых
операций со случайными (в диапазоне восьми бит) значениями операндов по десять раз. Для тех же занчений операндов 
вычисляется ожидаемый результат, после чего сравнивается с фактическим.

Функциональное покрытие: не проверяются ситуации, когда отсутствуют флаги валидности данных, а так же команды, не попадающие
в диапазон описанных. Иными словами, отсутствуют тесты с негативным сценарием.
